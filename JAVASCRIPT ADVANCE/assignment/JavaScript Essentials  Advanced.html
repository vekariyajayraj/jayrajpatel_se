If the hypothesis is true and the conclusion is false, then the conditional statement is false.

Likewise, if the hypothesis is false the whole statement is false.

Q.50 What is the drawback of declaring methods directly in JavaScript objects?

Memory Redundancy: Each object instance has its own copy of the method, leading to increased memory usage when multiple instances are created.

Inflexibility: It can be difficult to modify or extend the functionality across multiple instances since changes need to be made in every object.

Inheritance Complexity: Directly declared methods make it harder to manage inheritance and method overriding, complicating object-oriented design.

ontext Loss: When methods are used as callbacks, the `this` context may be lost, leading to unexpected behavior unless bound explicitly.

Performance Issues: Creating methods on each instance can slow down performance, especially when many objects are instantiated, as it consumes more memory and resources.

Q.60 What is Bom vs Dom in JS?
Feature BOM DOM
Definition Interacts with the browser window Represents the HTML document
key Objects window, navigator, location, history document, Element, Node
Purpose Control browser behavior and properties Manipulate HTML structure and content

BOM (Browser Object Model):

Definition: BOM refers to the objects provided by the browser that allow JavaScript to interact with the browser window and perform operations related to the browser itself.

Key Features:

Window Object: The global object that represents the browser window. It contains properties and methods related to the browser.

Navigator Object: Provides information about the browser (e.g., userAgent, platform).

Location Object: Contains information about the current URL and allows manipulation (e.g., changing the URL).

History Object: Allows navigation through the browser's session history (e.g., back and forward).



DOM (Document Object Model):

Definition: DOM is a programming interface for web documents, representing the structure of a webpage as a tree of objects. It allows JavaScript to manipulate the content, structure, and style of the webpage.

Key Features:

Represents the entire HTML document as a tree of nodes (elements, attributes, text).

Enables dynamic updates to the content and structure of the webpage.

Q.61 Array vs object defences in JS?
Feature Arrays Objects
Definition Ordered collection of elements Collection of key-value pairs
Access By numeric index By property name (key)
Order Maintains order of elements No guaranteed order
Methods Rich built-in methods (e.g., push, pop) Fewer built-in methods; can be extended
Usage List of items Complex entities with attributes
Example Access array[0] object.property or object['property']

Arrays

Definition: Arrays are ordered collections of data, where each element has a numeric index.


Characteristics:

Index-Based: Elements are accessed using their index, starting from 0.

Order: Elements maintain their order of insertion.

Methods: Arrays come with built-in methods for adding, removing, and manipulating elements (e.g., push, pop, shift, unshift, splice, slice).


Usage:

When you need an ordered list of items.

Suitable for iterating through elements.



Example:-


[ Apple , Banana , Cherry ]



Objects:

Definition: Objects are collections of key-value pairs, where the keys are strings (or symbols) and the values can be of any type.


Characteristics:

Key-Based: Properties are accessed using their keys.

Unordered: The properties do not have a guaranteed order.

Methods: Objects do not have as many built-in methods as arrays, but they can be extended with custom methods.


Usage:

When you need to store data with named properties.

Suitable for representing complex entities with various attributes.



Example:-



Use Arrays:

When you need an ordered list.

When the order of elements is important.

For collections of data where elements are accessed by position.


Use Objects:

When you need to store data with named properties.

When the relationship between data is more complex.

For entities with various attributes and behaviors.


Combining Arrays and Objects:

Sometimes, you may need to use arrays and objects together to represent more complex data structures.
